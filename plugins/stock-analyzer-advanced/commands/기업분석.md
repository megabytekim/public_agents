# 기업분석 명령어

이 커맨드는 **PI(Portfolio Intelligence) Orchestrator**를 호출하여 종합 기업 분석을 수행합니다.

## 실행 방법

```bash
/stock-analyzer-advanced:기업분석 [ticker] [depth]

# 예시
/stock-analyzer-advanced:기업분석 NVDA
/stock-analyzer-advanced:기업분석 삼성전자 deep
/stock-analyzer-advanced:기업분석 000660 quick
```

## 아키텍처

```
┌─────────────────────────────────────────────────────────┐
│                    PI (Orchestrator)                     │
│                planning-with-files 적용                  │
└─────────────────────────────────────────────────────────┘
                          │
          ┌───────────────┴───────────────┐
          │                               │
          ▼                               ▼
┌─────────────────────┐       ┌─────────────────────┐
│   MI (Worker)       │       │   SI (Worker)       │
│   • 시장 데이터     │       │   • 센티먼트        │
│   • 가격/뉴스/재무  │       │   • 종토방/Reddit   │
│   • 애널리스트 의견 │       │   • 이상징후 탐지   │
└─────────────────────┘       └─────────────────────┘
```

## 6-Phase 워크플로우

| Phase | 담당 | 내용 |
|-------|------|------|
| **1** | PI | 계획 수립, task_plan.md 생성 |
| **2a** | MI (Worker) | 가격, 뉴스, 재무 데이터 수집 |
| **2b** | SI (Worker) | 종토방/Reddit 센티먼트 수집 |
| **3** | PI | MI+SI 통합, 섹터 분석, Moat 평가 |
| **4** | PI | Entry/Exit 전략 수립 |
| **5** | PI | 최종 보고서 생성 및 저장 |

**Phase 2a + 2b는 병렬 실행**하여 시간 단축

## 분석 깊이

| 깊이 | MI 수집 | SI 수집 | planning-with-files |
|------|---------|---------|---------------------|
| `quick` | 가격 + 뉴스 2개 | 생략 | 선택 |
| `standard` | 전체 데이터 | 종토방/Reddit 스캔 | 권장 |
| `deep` | 경쟁사 비교 포함 | 심층 센티먼트 + 이상징후 | 필수 |

## 출력 파일

```
watchlist/stocks/[ticker]/
├── task_plan.md      # 진행 상황 추적
├── notes.md          # MI + SI 수집 데이터
└── [ticker]_분석.md  # 최종 보고서
```

## 보고서 구성

```markdown
1. 시장 데이터 (MI)      - 가격, 뉴스, 재무지표
2. 시장 센티먼트 (SI)    - 커뮤니티 의견, 스코어
3. 전략적 분석 (PI)      - 섹터, Moat, 테제
4. 투자 전략             - Entry/Exit
5. 리스크 요인           - 펀더멘털 + 센티먼트 리스크
6. 결론                  - MI vs SI 크로스체크 포함
```

## 실행

PI Orchestrator를 호출합니다:

```
[PI 에이전트 호출]

ticker: {{ticker}}
depth: {{depth}}

PI가 6-Phase 워크플로우를 실행하며:
1. planning-with-files 패턴 적용
2. MI Worker에게 시장 데이터 수집 위임
3. SI Worker에게 센티먼트 수집 위임 (병렬)
4. MI + SI 데이터 통합 분석
5. 섹터 지식 결합하여 전략 수립
6. 결과 저장
```
